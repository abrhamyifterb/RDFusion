Installation & Activation

Marketplace install

  1. Open VS Code → Extensions (Ctrl/Cmd + Shift + X).
  2. Search for “RDFusion” and click Install.

Install from VSIX

  1. Build or download rdfusion-x.y.z.vsix.
  2. VS Code → Extensions → ⋯ menu → Install from VSIX… → choose the file.

Activation triggers

The extension activates automatically when you open a .ttl (Turtle) or .jsonld (JSON-LD) file.

Requirements & Internationalization

  • Works on VS Code. Internet is recommended for things such as remote vocabulary lookups (LOV, prefix registries).
  • UI is English-only in this version.

Quick start

  1. Create or open a .ttl or .jsonld file.
  2. Type a known prefix (e.g., foaf:) or start authoring a triple. Autocomplete and validation appear as you type.
  3. Press Ctrl/Cmd + Space to force suggestions at any time, in case not triggered when it should.
  4. Open the Command Palette (Ctrl/Cmd + Shift + P) and run RDFusion commands like Filter Triples or Generate VoID.

Configuration & Settings

RDFusion exposes settings in File → Preferences → Settings (search for “RDFusion”) or via settings.json. These configurations are also available through the Command Palette.

Validation

  • Real-time Validation: on/off for live diagnostics; use the command palette for a quick toggle or through Settings.
  • SHACL: enable/disable applying SHACL shapes from your workspace.
  • Language Tags, xsd-Datatype, Duplicate, IRI Schemes, and Other validations.

Autocompletion

  • Auto-declare prefixes: when using an undeclared prefix, add the declaration automatically.
  • Suggestion Sources: choose local only, remote only, or both; either through the Command Palette or Settings.
  • Trigger Behavior: control whether suggestions appear automatically or only on Ctrl/Cmd + Space.

Performance

  • Validation Throttling: only enable validations needed to keep responsive in large files. e.g. SHACL

Real-time Validation

As you edit, the extension checks syntax, data consistency, and SHACL constraints and shows inline diagnostics.

Live error highlighting

Syntax errors and warnings are underlined as you type, with details shown inline.

SHACL constraints

If your workspace contains SHACL shapes (or one is configured), they are applied automatically if enabled.

Toggle & performance

Use Command-Palette or Settings to enable/disable checks. Basic parsing errors are prioritized; deeper checks run when syntax is corrected.

Configurable rules

See Configuration & Settings → Validation to enable/disable validations.

Autocompletion & Prefix Management

Triggering suggestions

  • Suggestions appear automatically in context; press Ctrl/Cmd + Space anytime.
  • In Turtle, typing prefix: (e.g., foaf:) shows terms.
  • In JSON-LD, editing @context or a compact IRI shows terms and ensures prefixes are declared.

Local & remote sources

Terms come from your open files and (optionally) remote vocabularies (e.g., FOAF via prefix registries / LOV). Remote results are cached.

(TTL/JSON-LD)

Provides based on whether you are inside a triple, a prefix declaration, a JSON-LD context, etc., and tailor suggestions accordingly.

Automatic prefix insertion

Accepting a completion for an undeclared prefix auto-adds @prefix (Turtle) or a context entry (JSON-LD) at the top of your file in a single undoable edit.

Refactor JSON-LD CURIEs

compact IRIs in your JSON-LD file. The refactor updates @context and rewrites all matching in the document.

Configurable behavior

Disable remote lookups when offline, and turn auto-declare/refactor options on/off in Settings based on your flow.

Three flows: Turtle completion, JSON-LD completion, and JSON-LD IRI compaction.
Example snippets (copyable)

Turtle

@prefix ex:  .
@prefix foaf:  .

ex:Homer a foaf:Person ;
  foaf:name "Homer Simpson" ;
  foaf:age 39 .

JSON-LD

{
  "@context": {
    "ex": "http://example.org/",
    "foaf": "http://xmlns.com/foaf/0.1/"
  },
  "@id": "ex:Homer",
  "@type": "foaf:Person",
  "foaf:name": "Homer Simpson",
  "foaf:age": 39
}

Triple Management Tools

Filter triples (by subject / predicate / object)

Use Command Palette → RDFusion: Filter Triples by … to generate a new tab view containing only matching triples. Useful for focusing on one resource (e.g., ex:Homer).

Click to view larger. Non-destructive: your original file is untouched.

Sort & group

Group by Subject groups triples under their subject; Sort Triples orders triples for cleaner diffs and review.

Format Turtle document

Use RDFusion: Format Document to apply consistent indentation, prefix-mapping, block wrapping, and so on.

IRI shortening (compact view)

Toggle visual shortening for long IRIs (e.g., http://example.org/.../Concept). This improves readability without changing file content.

Merge files

Select multiple RDF files and run RDFusion: Merge Files. The result is a single grouped Turtle document opened in a new tab.

Turtle RDF diff

Compare current open turtle file with the HEAD, or specific commit in Git. The diff view compares the current working, either with the HEAD git or the referenced commit.

Note: When blank nodes are present it shows RDF differences (support will be added soon).

Dataset statistics (VoID)

Run RDFusion: Generate VoID Description to compute counts (triples, distinct subjects/objects, classes) and output a VoID Turtle in a new tab.

Example VoID snippet 
<> a void:Dataset ;
				void:triples 123 ;
				void:distinctSubjects 45 ;
				void:distinctObjects 50 ;
				void:classes 10 ;
				void:classesPartition [ void:class ex:Person ; void:entities 5 ] .
